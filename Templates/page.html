Required

class in Core.Shared

/

Implemented in: Core.Shared.Utilities

Description

Force the user to assign an object to a field.

Unity still runs with empty or null fields. To avoid checks or runtime errors, add the [Required] attribute to them.

Unity will show a warning under the assigned field and will run a check before entering play mode. This is done to prevent NullReferenceException and to ensure that every field that requires a set reference is not empty.

To prevent anyone from entering playmode before all fields are assigned, Unity will check every object in the hierarchy that may use the [Required] attribute, checking that the attribute is not empty. If no error occurs playmonde will start without problems, otherwise playmonde won't start and the console will show log errors pointing at the exacts scripts that needs a reference. 

using UnityEngine;
using Core.Shared.Utilities

public class ExampleScript : MonoBehaviour
{
    //This field does not need a reference
    public float number = 20;

    //This field needs a reference for it to work as intended
    [Required] public Health healthScript;

    void OnCollisionEnter(Collider collider)
    {
        healthScript.Damage(number);
    }
}
